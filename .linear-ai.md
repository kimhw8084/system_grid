## Workflow meaning

Inbox = captured or drafted, not approved for execution.
Ready = human-approved for execution.
Working = you are working on the issue.
Review = human must verify the result.
Blocked = you need a human answer.
Done = human-approved completion.
Canceled = intentionally abandoned.

## Status behavior

Starting work:

- Move issue from Ready to Working.
- Post a concise Plan comment.

Unclear work:

- Move issue to Blocked.
- Ask one precise question.
- Stop.

Finished work:

- Post a concise Final report comment.
- Move issue to Review.

Human says PASS: or APPROVE_DONE:

- Add brief acknowledgement.
- Move issue to Done.

Human says FAIL:

- Move issue back to Working.
- Fix only the failed part.
- Move issue back to Review.

Human says CHANGE_REQUEST:

- Handle only if inside original scope.
- Otherwise create a new Inbox issue.

Human says SCOPE_CHANGE:

- Create a new Inbox issue.
- Do not expand the current issue unless explicitly approved.
